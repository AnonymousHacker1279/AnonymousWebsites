How to Make a Radio
Transmitter with Raspberry
Pi
Broadcast your music over the radio with a
Raspberry Pi
This tutorial was last updated on 9/26/19.
Have you ever wanted to play your music on the radio? You have probably seen small
devices which connect to your music player via a headphone jack and convert it to radio
signals. These can be useful in small areas, and they are usually effective for a few feet.
Sometimes though, you need some POWER in that broadcast! We mean forcing others
to listen to your music, because the frequency you select will override any radio stations
nearby.
Getting Supplies
You don’t need much for this project. The only variability in items will depend on how far
you want to broadcast audio. If you are only doing it for an example, and don’t need
more than a few feet, just get your Raspberry Pi. The model doesn’t matter, because we
will be using GPIO pins, which are present on all models. However, the image diagram
to show the GPIO setup is for the B+ and later models.
You don’t need an expensive setup if you want to
broadcast your music over FM radio. With this
project, you will be able to broadcast audio files up
to 200 feet around you!
If you want to broadcast for about 25 feet, a small piece of wire is fine. We recommend
using a female-to-male wire, so it attaches easily to the GPIO header.
Anything more than that typically requires an antenna. You can start with a wire as
mentioned above, then use an alligator clip to attach it to an antenna. It doesn’t need to
be powered. In testing, one of our developers was able to achieve about 200 feet of
distance between the Pi and the radio before the signal was lost.

Install Software
We will be using a GitHub library for this. At the terminal (accessible from the taskbar),
type “git clone https://github.com/markondej/fm_transmitter” This will download the
required files for the radio transmitter. Next, type “cd fm_transmitter”. This changes
directories into the one you just downloaded. Finally, type “make”, and the files will be
set up.
Using the Radio Transmitter
When you are ready to play music, type “sudo ./fm_transmitter -f 102.0
acoustic_guitar_duet.wav” to play the sample song at a 102 MHz frequency. You can
change the frequency and file, but only uncompressed WAVE (.wav) files work.
If you want to support more formats, you can install a program to convert it to WAVE,
such as SoX. To install SoX, type “sudo apt-get install sox libsox-fmt-mp3” to install SoX
and the MP3 libraries. When it is done, you can type “sox AUDIOFILE.mp3 -r 22050 -c
1 -b 16 -t wav - | sudo ./fm_transmitter -f 102.0 –“ and SoX will convert the file to WAV
and pipe it into the file input of the transmitter program. You can change some of the
settings, depending on your file, like bit number.
How it works
The reason this works is quite simple when you break it down. First, all processors have
something called a hardware clock. This determines the highest speed it can process
information at while still being stable. A GPIO pin attaches to that clock, meaning it can
put out radio signals. The program you installed allows you to push an audio file through
the clock at a certain frequency. In this case, the frequency can be heard on the FM
spectrum (88 MHz to 108 MHz).
